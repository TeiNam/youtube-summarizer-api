# 요구사항 문서

## 소개

유튜브 영상 URL을 입력받아 해당 영상의 내용을 자동으로 추출, 번역, 요약 및 정리하는 REST API이다. 자막이 있는 영상은 자막 데이터를 활용하고, 자막이 없는 영상은 음성 인식(Transcribe)을 통해 텍스트를 추출한다. 번역 및 요약 작업은 AWS Bedrock LLM을 활용하여 수행한다. 프레임워크는 FastAPI(Python)를 사용한다.

## 용어 정의

- **API_서버**: FastAPI 기반의 유튜브 요약 REST API 서버
- **자막_추출기**: 유튜브 영상에서 자막 데이터를 가져오는 모듈
- **음성_인식기**: 자막이 없는 영상의 오디오를 텍스트로 변환하는 모듈 (AWS Transcribe 활용)
- **요약_엔진**: AWS Bedrock LLM을 활용하여 텍스트 번역, 요약, 정리를 수행하는 모듈
- **유튜브_URL**: `https://www.youtube.com/watch?v=...` 또는 `https://youtu.be/...` 형식의 유효한 유튜브 영상 주소
- **요약_결과**: 번역된 텍스트, 요약문, 핵심 포인트 등을 포함하는 구조화된 응답 객체

## 요구사항

### 요구사항 1: 유튜브 URL 입력 및 검증

**사용자 스토리:** 사용자로서, 유튜브 URL을 API에 전달하여 영상 요약을 요청하고 싶다.

#### 인수 조건

1. WHEN 유효한 유튜브_URL이 POST 요청으로 전달되면, THE API_서버 SHALL 해당 요청을 수락하고 처리를 시작한다
2. WHEN 유효하지 않은 형식의 URL이 전달되면, THE API_서버 SHALL HTTP 422 상태 코드와 함께 URL 형식 오류 메시지를 반환한다
3. WHEN URL이 빈 문자열이거나 누락된 경우, THE API_서버 SHALL HTTP 422 상태 코드와 함께 필수 필드 누락 메시지를 반환한다
4. THE API_서버 SHALL `https://www.youtube.com/watch?v=` 및 `https://youtu.be/` 형식의 유튜브_URL을 지원한다
5. WHEN 존재하지 않는 유튜브 영상의 URL이 전달되면, THE API_서버 SHALL HTTP 404 상태 코드와 함께 영상을 찾을 수 없다는 메시지를 반환한다

### 요구사항 2: 자막 추출

**사용자 스토리:** 사용자로서, 자막이 있는 유튜브 영상의 자막 데이터를 자동으로 추출하고 싶다.

#### 인수 조건

1. WHEN 유효한 유튜브_URL이 전달되면, THE 자막_추출기 SHALL 해당 영상의 자막 존재 여부를 확인한다
2. WHEN 자막이 존재하는 영상이면, THE 자막_추출기 SHALL 자막 텍스트를 추출하여 반환한다
3. WHEN 여러 언어의 자막이 존재하면, THE 자막_추출기 SHALL 원본 언어 자막을 우선적으로 선택한다
4. IF 자막 추출 과정에서 오류가 발생하면, THEN THE 자막_추출기 SHALL 오류 내용을 로그에 기록하고 음성_인식기로 처리를 위임한다

### 요구사항 3: 음성 인식을 통한 텍스트 추출

**사용자 스토리:** 사용자로서, 자막이 없는 유튜브 영상도 음성 인식을 통해 내용을 추출하고 싶다.

#### 인수 조건

1. WHEN 자막이 없는 영상이 감지되면, THE 음성_인식기 SHALL 해당 영상의 오디오를 다운로드한다
2. WHEN 오디오 다운로드가 완료되면, THE 음성_인식기 SHALL AWS Transcribe를 사용하여 오디오를 텍스트로 변환한다
3. IF 오디오 다운로드에 실패하면, THEN THE API_서버 SHALL HTTP 502 상태 코드와 함께 오디오 다운로드 실패 메시지를 반환한다
4. IF 음성 인식 처리에 실패하면, THEN THE API_서버 SHALL HTTP 502 상태 코드와 함께 음성 인식 실패 메시지를 반환한다
5. WHEN 음성 인식이 완료되면, THE 음성_인식기 SHALL 변환된 텍스트를 반환한다

### 요구사항 4: 텍스트 번역

**사용자 스토리:** 사용자로서, 추출된 텍스트를 원하는 언어로 번역하고 싶다.

#### 인수 조건

1. WHEN 텍스트와 대상 언어가 전달되면, THE 요약_엔진 SHALL AWS Bedrock LLM을 사용하여 텍스트를 대상 언어로 번역한다
2. THE API_서버 SHALL 요청 시 대상 언어를 지정할 수 있는 파라미터를 제공한다
3. WHEN 대상 언어가 지정되지 않으면, THE API_서버 SHALL 기본 대상 언어를 한국어(ko)로 설정한다
4. IF 번역 처리 중 AWS Bedrock 호출에 실패하면, THEN THE API_서버 SHALL HTTP 502 상태 코드와 함께 번역 서비스 오류 메시지를 반환한다

### 요구사항 5: 텍스트 요약 및 정리

**사용자 스토리:** 사용자로서, 영상 내용을 간결하게 요약하고 핵심 포인트를 정리하여 받고 싶다.

#### 인수 조건

1. WHEN 번역된 텍스트가 준비되면, THE 요약_엔진 SHALL AWS Bedrock LLM을 사용하여 텍스트를 요약한다
2. THE 요약_엔진 SHALL 요약_결과에 전체 요약문을 포함한다
3. THE 요약_엔진 SHALL 요약_결과에 핵심 포인트 목록을 포함한다
4. IF 요약 처리 중 AWS Bedrock 호출에 실패하면, THEN THE API_서버 SHALL HTTP 502 상태 코드와 함께 요약 서비스 오류 메시지를 반환한다

### 요구사항 6: API 응답 형식

**사용자 스토리:** 사용자로서, 구조화된 JSON 형식으로 요약 결과를 받고 싶다.

#### 인수 조건

1. THE API_서버 SHALL 모든 성공 응답을 JSON 형식으로 반환한다
2. THE API_서버 SHALL 응답에 영상 제목, 원본 언어, 번역된 텍스트, 요약문, 핵심 포인트를 포함한다
3. THE API_서버 SHALL 모든 오류 응답에 오류 코드와 오류 메시지를 포함한다
4. THE API_서버 SHALL 텍스트 추출 방식(자막 또는 음성 인식)을 응답에 포함한다

### 요구사항 7: 비동기 처리 및 상태 조회

**사용자 스토리:** 사용자로서, 긴 처리 시간이 소요되는 요약 작업의 진행 상태를 확인하고 싶다.

#### 인수 조건

1. WHEN 요약 요청이 접수되면, THE API_서버 SHALL 작업 ID를 즉시 반환하고 백그라운드에서 처리를 진행한다
2. WHEN 작업 ID로 상태 조회 요청이 들어오면, THE API_서버 SHALL 현재 처리 단계(대기중, 텍스트 추출중, 번역중, 요약중, 완료, 실패)를 반환한다
3. WHEN 작업이 완료되면, THE API_서버 SHALL 상태 조회 응답에 요약_결과를 포함한다
4. IF 존재하지 않는 작업 ID로 조회 요청이 들어오면, THEN THE API_서버 SHALL HTTP 404 상태 코드와 함께 작업을 찾을 수 없다는 메시지를 반환한다

### 요구사항 8: 오류 처리 및 로깅

**사용자 스토리:** 운영자로서, API의 오류 상황을 추적하고 디버깅할 수 있도록 로깅이 되어야 한다.

#### 인수 조건

1. THE API_서버 SHALL 모든 요청과 응답을 구조화된 형식으로 로그에 기록한다
2. IF 예상치 못한 오류가 발생하면, THEN THE API_서버 SHALL HTTP 500 상태 코드와 함께 내부 서버 오류 메시지를 반환한다
3. THE API_서버 SHALL 오류 발생 시 스택 트레이스를 로그에 기록한다
4. IF AWS 서비스(Bedrock, Transcribe) 호출 시 타임아웃이 발생하면, THEN THE API_서버 SHALL HTTP 504 상태 코드와 함께 서비스 타임아웃 메시지를 반환한다
